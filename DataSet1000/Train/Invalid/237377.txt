input boxes do not show value if name="today" and size, maxlength defined
User-Agent:       Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.5) Gecko/20031007 Firebird/0.7
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.5) Gecko/20031007 Firebird/0.7

Values of input boxes are not shown inside of the boxes when the name="today"
and maxlength, size attributes are specified.

Reproducible: Always
Steps to Reproduce:
1. Put <input type="text" value="3" name="today" maxlength="3" size="3" /> into
a page.

Actual Results:  
The value attribute, 3, is not displayed in the input box.

Expected Results:  
Should have shown a 3 in the input box.

Workaround: Don't use "today" as the value of a name attribute.
