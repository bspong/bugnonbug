TBODY tag ignores width, 'jumps' around if display property is updated by javascript
User-Agent:       Mozilla/4.0 (compatible; MSIE 6.0; Windows NT 5.1; IE 6.0 SP1 (FID r3.0); .NET CLR 1.0.3705; .NET CLR 1.1.4322; InfoPath.1)
Build Identifier: 1.5.0.1

If multiple TBODY tags are used within a table definition and the style display attribute for each TBDOY is set to none, the TBODY does not render correctly. When combined with a JavaScript to enable paging, the TBODY 'jumps' around.
The setup described above renders and functions correctly in IE but not in Firefox. 

Reproducible: Always

Steps to Reproduce:
1.See Details for example


Expected Results:  
TBODY should render with specified width tag.
