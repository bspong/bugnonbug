different ports, 80 and 8080. The same contents is returned, of course; but it
is displayed differently by Mozilla. With port 8080 it is displayed with no
highlighting at all; it is correctly displayed when retrieved on port 80 or as a
local file.

Reproducible: Always

Steps to Reproduce:
1. Run tomcat on port 8080, browse to the file, see the behavior.
2. Run tomcat on port 80, browse to the file, see the behavior.
3. Open the file with Mozilla, see the behavior.

Actual Results:  
XML file displayed as plain text when opened on port 8080.

Expected Results:  
1. The source ought to be beautifully hightlighted, as it is when the file is
opened locally.
