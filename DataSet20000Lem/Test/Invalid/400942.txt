Mouse on Tab generate Network traffic and cooky reload
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.0; en-US; rv:1.8.1.8) Gecko/20071008 Firefox/2.0.0.8
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.0; en-US; rv:1.8.1.8) Gecko/20071008 Firefox/2.0.0.8

Mouse movement on tab (from one to another) whithout clicking generate Network traffic and cooky reload for the tab under pointer.
Which is quite anoyning.

Reproducible: Always

Steps to Reproduce:
1.Open at least 2 tab
2.One of the site should have cooky and Firefox should be configured to ask for cooky on this site
3.Click on one different tab (not the one with cookies)
4.Move the mouse pointer from one tab to another
Actual Results:
Network traffic is generated for each tab under mouse pointer
If site have cooky and Firefox is configured to ask permission then you get all cooky permission window

Expected Results:
No Network traffic and cooky permission when mouse is moved over tabs.
