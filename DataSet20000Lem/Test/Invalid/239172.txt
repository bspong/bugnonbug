When using Firefox is on a secondary monitor menu open onto the primary monitor.
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.6) Gecko/20040206 Firefox/0.8
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.6) Gecko/20040206 Firefox/0.8

When Firefox is on a secondary monitor clicking on the menu title brings the
menu up on the primary monitor.

If the second monitor is to the left of the main one the menu's y position is
correct, but it x position place the left side of the menu on the left side of
the primary monitor.

If the second monitor is to the right of the main one the menu's y position is
correct, but it x position place the right side of the menu on the right side
of the primary monitor.

If the second monitor is above of the main one the menu's x position is correct,
but it y position place the top side of the menu on the top side of the
primary monitor.

If the second monitor is above of the main one the menu's x position is correct,
but it y position place the bottom side of the menu on the bottom side of the
primary monitor.

This problem also occurs with the pull down for the address entry box

Reproducible: Always
Steps to Reproduce:
1. Start Mozilla Firefox
2. Move Firefox onto the secondary monitor
3. Click on the Bookmarks menu
Actual Results:
The menu wa brought up on the primary monitor.

Expected Results:
The menu should have been brought up on the secondary monitor with the menu
left side left justified to the menu's title on the menu bar.
