Incorrect error message for non-existent sites.
User-Agent: Mozilla/5.0 (X11; U; Linux x86_64; en-US; rv:1.9.1.3) Gecko/20090909 Fedora/3.5.3-1.fc11 Firefox/3.5.3 GTB5
Build Identifier: Mozilla/5.0 (X11; U; Linux x86_64; en-US; rv:1.9.1.3) Gecko/20090909 Fedora/3.5.3-1.fc11 Firefox/3.5.3 GTB5

If I visit a non-existent URL, such a http://noexist.noexist.noe, instead of getting an error that Firefox cannot find the server, I get an empty error message, with a non-functional "Or you can add an exception…" link and a "Try Again" button.

This only happens of the host name in the URL is of the form <hostname>.<domainname>.<tld>, a in the example above. If it is in the form <domainname>.<tld>, e.g., noexist.noe, I get the correct error message.

Reproducible: Always

Steps to Reproduce:
1. Visit http://noexist.noexist.noe (I use the non-existent "noe" TLD, to make sure that the URL stay non-existent.)
Actual Results:
Empty error message, with a non-functional "Or you can add an exception…" link and a "Try Again" button.

Expected Results:
A dialog saying that Firefox can't find the server at noexist.noexist.noe.

I have seen this error under Linux and Windows XP.
