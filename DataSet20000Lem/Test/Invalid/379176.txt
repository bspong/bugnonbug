Curly brackets, "{", in Courier New and monospace font are knocked down a pixel
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.8.1.3) Gecko/20070309 Firefox/2.0.0.3
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.8.1.3) Gecko/20070309 Firefox/2.0.0.3

When using the Courier New or monospace fonts, the opening curly bracket symbol, {, is positioned 1 pixel lower than it should be, even if the line-height ha been set. It is a though the padding top is 1 pixel more than it's meant to be.

Reproducible: Always

Steps to Reproduce:
1. Create a page that us the the Courier New font (line-height: 16px;)
2. Type lot of line of text, some of which contain "{"s
3. Select the text, print screen, paste into an image editor and look at the spacing between the line
4. You will see that there is a larger gap between the line with the "{" and the line above it, than there is between any other two lines.
Actual Results:
The gap between the line with the "{" on it and the line above is 1px too large.

Expected Results:
The gap between the line with the "{" on it and the line above should be the same a the rest of the lines.
