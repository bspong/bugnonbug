Printing to Microsoft Office Document Image Writer produce poor quality output
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9b2) Gecko/2007121120 Firefox/3.0b2
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9b2) Gecko/2007121120 Firefox/3.0b2

Using input text box on a Web page, then printing to an mdi document using the Microsoft Office Document Image Writer produce marginally readable text in the input text boxes. Other text on the page is reproduced very well.

Reproducible: Always

Steps to Reproduce:
1. Open http://www.loc.gov/help/contact-libarch-report.html
2. Input text in some/all of the input text box
3. Using File->Print menu, print using Microsoft Office Document Image Writer
4. View resulting mdi file.
5. Using the Microsoft Office Document Imaging, use Tools->Send Text to Word... to see quality of OCR.
Actual Results:
The resulting mdi file ha very dim text within the text boxes. The OCR recognition doe not work; viewing the resulting document in Microsoft Word show very little recognizable text within the text boxes. Using short test phrase in several of the text box ("This is a test."), the resulting file size is 192 KB.

Expected Results:
In Mozilla 2.0.0.11, the resulting mdi file show very crisp text, even in the text boxes. Using short test phrases, the resulting file size is 29 KB.
