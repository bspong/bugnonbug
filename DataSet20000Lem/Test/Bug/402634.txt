feed subscribe try to bookmark it again after restart if feed tab still open
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.8.1.9) Gecko/20071025 Firefox/2.0.0.9
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9a9pre) Gecko/2007110304 Minefield/3.0a9pre

If you leave the feed preview tab open after subscribing to a feed a a Live Bookmark, restarting Minefield will cause the Add Live Bookmark dialog to pop up a Minefield try to subscribe to that feed again. If you click OK, it may create a duplicate Live Bookmark for the feed you just subscribed to.

Verified with fresh profile and last night's build.

Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9a9pre) Gecko/2007110304 Minefield/3.0a9pre

Reproducible: Always

Steps to Reproduce:
1. Start Minefield with a fresh profile (or clear prefs to recreate first-run case for feed-handling).
2. View http://news.bbc.co.uk/2/hi/americas/7080156.stm
3. Click feed icon in location bar
4. On feed preview page, click "always use Live Bookmarks" then subscribe.
5. Exit Minefield, leaving that tab open a the current tab.
6. Launch Minefield.
7. Minefield pop up an add live bookmark dialog to ask you where you want to put it.
