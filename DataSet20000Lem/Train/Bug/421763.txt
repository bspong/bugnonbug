bookmark import and restore only allow file named .html and .json

Build Identifier: Mozilla/5.0 (X11; U; Linux i686; en-US; rv:1.9b5pre) Gecko/2008030804 Minefield/3.0b5pre

When you choose to restore bookmark from a bookmark backup, it display and let you to select only file that have a particular extension, html.

You should be able to choose to display and select file that are not html. Although naming the file *.html is the normal way, and a good default to have, why shouldn't you be able to name your backup however you'd like?

--

Reproducible: Always

Steps to Reproduce:
1.Open the Library via Organize Bookmarks in the Bookmarks menu
2.From Import and Backup, choose Restore --> Choose file.
3.A dialogue pop up, and it (sensibly) display file with the suffix .html in selected directory
4.There is a box that allows you to choose what type of file are shown, but it le sensibly allows only one choice, html.
Actual Results:
You can't choose your bookmark backup if it is not named *.html

Expected Results:
Allow you to choose a backup with any kind of name or suffix.

The workaround is to rename your file a needed.
