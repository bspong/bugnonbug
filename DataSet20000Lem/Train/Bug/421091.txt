Bookmark menu doesn't close after hovering over it with favicon from location bar
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9b5pre) Gecko/2008030405 Minefield/3.0b5pre
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9b5pre) Gecko/2008030405 Minefield/3.0b5pre

All while holding a url icon - if after grabbing the url icon to make a bookmark you hover over the bookmark menu to open it then leave the area, the bookmark menu doesn't close. Even after dropping the url icon, the bookmark menu still stay open. Now all the menu can be switched to if you hover over them a if the menu system never closed.

Reproducible: Always

Steps to Reproduce:
1. Browse to any page
2. Grab the url icon a if to drag and drop a bookmark
3. Hover over the 'Bookmarks' menu
4. Leave the bookmark menu area

Actual Results:
Bookmarks menu stayed open until some other 'specific' action wa taken such a hitting the escape key(a second time if the escape key wa used to cancel the drag and drop) or clicking on a menu / page.

Expected Results:
After leaving the 'bookmarks' menu area with a drag and drop link, the bookmark menu should close.

This bug wa confirmed by other people in the firefox build forum on window vista.
