Attempting to drag a tab onto another application creates a new window instead
User-Agent: Mozilla/5.0 (Macintosh; U; Intel Mac OS X 10.5; en-GB; rv:1.9.1b2) Gecko/20081201 Firefox/3.1b2
Build Identifier: Mozilla/5.0 (Macintosh; U; Intel Mac OS X 10.5; en-GB; rv:1.9.1b2) Gecko/20081201 Firefox/3.1b2

Many application in OS X are designed to allow you to drag and drop thing in various ways, including dragging and dropping URLs from browser (or other applications) into place where either an HTML link, the text URL itself, or a link with the title of the page a the text would be useful. For example, if a user wish to share a page being read (or a Twitter tweet, which I use a a reference in my demonstration movie) with someone via instant message.

Reproducible: Always

Steps to Reproduce:
1. Click and hold title in tab
2. Drag from Firefox window onto input area of other app
3. Firefox creates a new window containing the tab


Expected Results:
Expected to see:

1. Click and hold title in tab
2. Drag from Firefox window onto input area of other app
3. A link (either the title of the tab [ideally], or the text URL itself [as wa the case in the Twitterific reference movie], should appear in the input area of the target app
