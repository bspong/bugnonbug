Handle exited add-on actor when reloading via remote debugger
STR
- Start Firefox
- Using the remote debugger, install an add-on temporarily (actor support landed in bug 1273183)
- Trigger a reload with the debug client (actor support landed in bug 1246030)
- Wait about 20 seconds. During this time, some add-on will be automatically uninstalled due to compatibility (I think).
- Trigger another reload

What should happen: The reload should work just fine. The add-on that get uninstalled are not needed for any part of the reload process.

What actually happens: You get a stack trace which look to be caused by the uninstalled add-ons.
