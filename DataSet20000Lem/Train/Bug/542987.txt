proxy-authorization header is re-sent on authenticated socket
User-Agent: Mozilla/4.0 (compatible; MSIE 8.0; Windows NT 5.1; Trident/4.0; .NET CLR 2.0.50727; .NET CLR 1.1.4322; .NET CLR 3.0.4506.2152; .NET CLR 3.5.30729; InfoPath.2)
Build Identifier: User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; de; rv:1.9.1.3) Gecko/20090824 Firefox/3.5.3 (.NET CLR 3.5.30729)

It seems that when Firefox get a 3xx HTTP status code back, it re-sends the previous proxy-authorization header with the NTLMSSP_NEGOTIATE (type 1) message.

This is incorrect behavior, once a proxy connection is authenticated (NEGOTIATE, CHALLENGE, AUTH), no further NTLMSSP header should be sent on this connection.

Its maybe somewhat similar or related to the bug "tree" 475053, but I believe this specific behavior ha not been discussed yet.
