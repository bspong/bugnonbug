Windows with App Tabs always restore on top/active -- despite not being on top when browser closed
User-Agent: Mozilla/5.0 (Windows NT 5.1; rv:2.0b4pre) Gecko/20100815 Minefield/4.0b4pre
Build Identifier: Mozilla/5.0 (Windows NT 5.1; rv:2.0b4pre) Gecko/20100815 Minefield/4.0b4pre

If you have two or more windows, with one of them contains one or more App Tabs, and you close Firefox from a window that doe not contain any App Tabs, on session restore, the App Tab-containing window is always forced on top/active.


Reproducible: Always

Steps to Reproduce:
1. Start Minefield with a new profile
2. Configure startup of: "Show my window and tab from last time"
3. Open another window, so you have at least two browser window
4. Create App Tabs in one of the window
5. Switch to a window that doe not contain any App Tabs.
6. File --> Exit to close Minefield
7. Relaunch the Minefield profile.
8. Notice that the Window containing the App Tab(s) always restore on top/active.
9. Compare with session restore behavior when there are no App Tabs defined.

Actual Results:
App Tabs window restore always on top.


Expected Results:
Whatever window wa on top/active at close should be on top on session restore, a it ha been without and before App Tabs.
