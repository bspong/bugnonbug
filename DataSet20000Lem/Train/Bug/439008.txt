The input "myinput." contains a period which is valid according to
http://www.w3.org/TR/html401/types.html#type-cdata

The POST return "myinput_". The period changed to an underscore.

<form action="/test" method="post">
<input name="myinput." type="text" value="mydata" />
<input name="submit" type="submit" value="Submit" />
</form>

POST returns:
array(2) {
["myinput_"]=>
string(6) "mydata"
["submit"]=>
string(6) "Submit"
}

"myinput." !== "myinput_"

The period changed to an underscore.

Reproducible: Always

Steps to Reproduce:
1.The "/test.php" page contains var_dump($_POST);
2.Submit the form

<form action="/test.php" method="post">
<input name="myinput." type="text" value="mydata" />
<input name="submit" type="submit" value="Submit" />
</form>

3. page.php will show that the form data submitted doe not match. The period changed to an underscore.
Actual Results:
myinput_

Expected Results:
myinput.
