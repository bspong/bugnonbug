Not all visited article from a website are added in "Spotlight" area even if they have the same page structure
[Affected versions]:
- Firefox 45.0 and up
- Activity Stream 1.0.4

[Affected Platforms]:
- All Windows
- All Mac OS
- All Linux

[Prerequisites]:
- Latest Activity Stream add-on version (1.0.4) wa previously installed on a clean profile.

[Steps to reproduce]:
1. Start Firefox browser with the profile where Activity Stream add-on wa previously installed.
2. Navigate to a website that ha multiple article with the same structure (eg: http://adevarul.ro).
3. Open a new tab, and wait for the "Top Sites" area to be populated with the visited website (additionally, open the "History" tab with Ctrl+h keys)
4. From the opened website tab, navigate to a different article from the page.
5. Refresh the "New Tab" page and observe the "Spotlight" area.
6. Repeat step #4 and #5 for several article and each time observe the "Spotlight" area.

[Expected results]:
Each of the visited website article are correctly displayed in "Spotlight" area and when a new one appears, the one from last position (right) is moved in "Top Activity" area.

[Actual results]:
Only some of the visited website article are displayed in "Spotlight" area even if they have the same page structure (title, image and text arranged the same).

[Notes]:
- Attached a screen recording with the issue while visiting " http://adevarul.ro" website articles.
- This behavior wa also observed for other websites, one of them is the "New York Times" website.
