'feed-icon-image' is not displayed for website even if it is available
Created attachment 8738907
Untitled.png

[Affected versions]:
- Firefox 45.0 and up
- Activity Stream 1.0.7

[Affected platforms]:
- All Windows
- All Mac OS
- All Linux

[Prerequisites]:
- Latest Activity Stream add-on version (1.0.7) wa previously installed on a profile.
- Have more website in "Recent Activity" area from Activity Stream New Tab.

[Steps to reproduce]:
1. Start Firefox browser with the profile where Activity Stream add-on wa previously installed.
2. Open a new tab and go to http://store.steampowered.com/app/380220/.
3. From the previously visited link, visit another URL that point to a game.
4. Repeat step 3 about three-four times.
5. Open a new tab and observe the "Recent Activity" section (if the visited website are not displayed, please open another New Tab).

[Expected result]:
- The visited website have displayed the feed-icon-image and site-icon-favicon.

[Actual result]:
- This issue ha a random order.
- Some of the visited website have displayed only the site-icon-favicon without feed-icon-image, although feed-icon-image exists.

[Notes]:
- This issue is also reproducible in Activity Stream Timeline.
- There is a task in GitHub (https://github.com/mozilla/activity-streams/issues/193) regarding this issue, but it is marked a Closed. This issue is still reproducible in Activity Stream v1.0.7.
