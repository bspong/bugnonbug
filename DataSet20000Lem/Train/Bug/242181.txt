Bookmark pop-out secondary menu not in synk with first menu entry on second monitor
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.6) Gecko/20040206 Firefox/0.8
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.6) Gecko/20040206 Firefox/0.8

When Firefox is on the second monitor and that monitor vertical alignment is
not the same a first monitor then the bookmark pop-out secondary menu is not
in sync with the first menu entry. This happens if the secondary menu doe not
fit between the monitor lower border and the fir menu entry.



Reproducible: Always
Steps to Reproduce:
1. set up a secondary monitor with "Extend my Windows desktop onto this monitor"
2. drag the monitor so that the top is not aligned with the first monitor
3. start Firefox and move it to the second monitor
4. move/add bookmark so that the bookmark doe not fit on the screen
5. add a bookmark-folder "Test", with some bookmarks, last on the bookmark list
6. click on menu "Bookmarks"
7. scroll down to bookmark-folder "test" and then the pop-menu with the
bookmark that are in the folder pop way above the menu-entry "test"


Actual Results:
the pop-menu with the bookmark that are in the folder pop way above the
menu-entry "test"

Expected Results:
the pop-menu with the bookmark that are in the folder should pop exactly above
the menu-entry "test"
