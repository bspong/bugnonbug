User-Agent: Mozilla/5.0 (Windows; U; Windows NT 6.0; en-GB; rv:1.9.2.12) Gecko/20101026 Firefox/3.6.12 ( .NET CLR 3.5.30729; .NET4.0E)
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 6.0; en-GB; rv:1.9.2.12) Gecko/20101026 Firefox/3.6.12 ( .NET CLR 3.5.30729; .NET4.0E)

The only way I can force Firefox to flush a Silverlight xap plugin from exitin the cache is by going to Tools -> Clear Recent History -> Uncheck all option except "Cache" and select "Last Hour".

This is tedious.

Reproducible: Always

Steps to Reproduce:
1. Create a Silverlight xap plugin
2. Deploy the xap plugin to a website
3. Load it in Firefox
4. Change the code and redeploy the xap plugin
5. Refresh the page using any of the "Hard" or "Forced" refresh method commonly used


Actual Results:
1. The cache wa not invalidated and Firefox did not request a new XAP plugin file.

Expected Results:
1. Firefox invalidates the cache and request a new XAP plugin file.

I have tested this using a XAP on localhost, but not on the general internet.

I have a workaround, but it is time consuming:
Go to Tools -> Clear Recent History -> Uncheck all option except "Cache" and select "Last Hour" for the time period, click OK to force your cache to be cleared using (likely) a different code path
