touch event are not fired on Win8 when requirejs is used
Created attachment 8404810
touch_requirejs.txt

User Agent: Mozilla/5.0 (Windows NT 6.1; WOW64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/33.0.1750.154 Safari/537.36

Steps to reproduce:

The following example is used to reproduce the bug on Windows 8.

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/1999/REC-html401-19991224/loose.dtd">

<html lang="en">
<head>
<meta charset="utf-8">
<title>test touch</title>

<script type="text/javascript" src="./require.js"></script>
</head>

<body>

<script>
require({}, [],

function () {

document.addEventListener('mousedown', function(){
console.log("mouse down");
});

document.addEventListener('touchstart', function(){
console.log("touch start");
});
}
);
</script>
</body>
</html>



Actual results:

Touch event are not fired a it should be.
When switching tab or windows, the touch event are recovered.
If addEventListener on touchstart is done outside of the callback called by require(), it works.


Expected results:

touch event should be fired consistently.
