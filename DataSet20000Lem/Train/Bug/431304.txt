tree-splitter doesnt work in dont-build-content rdf-templated tree loaded from xbl
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-GB; rv:1.9b5) Gecko/2008032620 Firefox/3.0b5
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-GB; rv:1.9b5) Gecko/2008032620 Firefox/3.0b5

I have several tree with flags='dont-build-content' set which load their data from a rdf datasource. The tree are defined in xbl.
The tree-splitters stopped working (can't adjust the column width a if there isn't any splitter defined) in Firefox 3b5 (does work in Firefox 2).
The tree-splitters work fine if I remove the dont-build-content flag, or set the ordinal index hardcoded on the treecolumns and splitter (Both thing I can't do for obvious reasons).

Reproducible: Always

Steps to Reproduce:
1. Define a xul window containing the reference to a test xbl file containing the tree content (see additional information for an example)
2. load the xul window in firefox
3. try adjusting the width of any column
Actual Results:
You cant adjust the column width. It look like if there wasn't any splitter defined.

Expected Results:
Expected to be able to adjust the width of the column

xul example
-----------------------------------------------------
<?xml version="1.0"?>
<?xml-stylesheet href="chrome://global/skin" type="text/css"?>
<?xml-stylesheet href="/Kiddo/Styles/Default.xul.css" type="text/css"?>

<window xmlns="http://www.mozilla.org/keymaster/gatekeeper/there.is.only.xul" xmlns:html="http://www.w3.org/1999/xhtml">
<box id="xx" class="test" style="-moz-binding:url('test.xbl.xml#test')"/>
</window>
