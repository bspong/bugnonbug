Input field with id="NombreEnfantEmpr" ha an event listener that cause all element bearing the "enfant" class to switch from display:"hidden" to display:"" (should be "run-in", but it isn't implemented in all browsers). Of those, the LI element that contains the id="AgeEnfantEmpr" imput field, which appears in the DOM after the LI element that contains the input field with id="NombreEnfantEmpr", instead appears above it.



Reproducible: Always

Steps to Reproduce:
1.Enter value > 0 in field with legend "Nombre d'enfants mineurs à charge"
2.See field bearing the legend "Ages de enfants mineurs à charge" appear above it
3.(optional) deactivate then reactivate styles, and now the field marked in 2. appears in the position it occupies in DOM.
Actual Results:
field bearing the legend "Ages de enfants mineurs à charge" appears first in "Situation de Famille" fieldset

Expected Results:
field bearing the legend "Ages de enfants mineurs à charge" should appear last in "Situation de Famille" fieldset

This bug is present in both Firefox 3.0 and 3.5 (Win32/Lin32/Lin64), but not in IE 8, Opera 9.6, Konqueror 4.2.2 or Safari 4.
