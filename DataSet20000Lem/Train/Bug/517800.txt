Crash/freeze when restoring large session
User-Agent: Mozilla/5.0 (compatible; Konqueror/4.3; Linux) KHTML/4.3.1 (like Gecko) Fedora/4.3.1-3.fc11
Build Identifier: Mozilla/5.0 (X11; U; Linux i686; en-US; rv:1.9.1.3) Gecko/20090909 Fedora/3.5.3-1.fc11 Firefox/3.5.3

After leaving Firefox running with 2 windows, 135 tab open it eventually crashes. This ha become expected behaviour.

However attempting to restore a session after the crash with Tab Mix Plus session manager cause the browser to become fragile while the page are loading. After testing, it appears that about 45% of the time the browser will crash outright with a segmentation fault when about halfway through loading the tabs, 45% of the time it will freeze at about the same point (reporting 0% cpu usage), and about 10% of the time it will restore successfully. Once the tab are all loaded the browser becomes stable.

Addons installed:
Flashblock
Tabhunter
Tab Mix Plus
Tree Style Tab

Plugins Installed:
Citrix Receiver 11.0.140395
MozPlugger 1.12.1
Shockwave Flash 10.0 r32
Skype Buttons for Kopete
Xine Plugin 1.0.2



Reproducible: Sometimes

Steps to Reproduce:
1. Leave session with large number of tab open for several day
2. Wait for application to crash, or exit manually.
3. Attempt to restore session with Tab Mix Plus.
Actual Results:
Browser start up inconsistently, either by crashing, freezing or starting up successfully.
