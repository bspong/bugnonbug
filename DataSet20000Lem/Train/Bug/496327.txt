Bonjour,

Les ligne de code HTML suivantes :

<div style='background: url();'></div>
<img src='' />
<script type='text/javascript' src=''></script>
<link rel='stylesheet' href='' type="text/css" />

Relance un appel vers la page dans laquelles elles sont appelées si l'information dans le attributs href, src ou la propriete url() est manquante.

Si de session sont utilisées dans la page en question, un freeze de firefox peu être constaté. Ce freeze est aléatoire et semble être sensible à la vitesse de connexion internet ou à la puissance du processeur de la machine. (bug constaté à 99% sou Firefox Mac)

Il se peu que cela soit du au faite que la page n'ait pa encore fini de charger côté serveur lorsque Firefox relance un appel à la page. du coup la session est relancé, mais bloqué car non libéré par le script parent. Ce qui pourrait posé ce problème.

Bien entendu il n'y a aucune raison que ce tag soient saisi ainsi dans une page HTML. Mais je trouve ce rappel assez étrange.

Reproducible: Always

Steps to Reproduce:
Dans un script PHP
<?php
session_start();
?>
<html>
<head>
<link rel='stylesheet' href='' type="text/css" />
<script type='text/javascript' src=''></script>
</head>
<body>
<img src='' />
<div style='background: url();'>test</div>
</body>
</html>
<?php
session_write_close();
?>
Actual Results:
Avec firebug, vous devriez voir apparaitre 4 appels à la page dans lequel ce code source est enregistré.

Si vous êtes sou mac, et actualiser plusieurs fois cette page, vous pouvez avoir un freeze du à la fonction session_start();
