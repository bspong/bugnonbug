PDF upload via ASP.Net FileUpload component return MIME "application/unknown"
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 6.0; en-US; rv:1.9.1.3) Gecko/20090824 Firefox/3.5.3 (.NET CLR 3.5.30729)
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 6.0; en-US; rv:1.9.1.3) Gecko/20090824 Firefox/3.5.3 (.NET CLR 3.5.30729)

I have an ASP.Net FileUpload control on an aspx. Once compiled it turn into an html input type="file" control.

When I browse to a PDF and then click the Upload button I attempt to extract the MIME type based on the PostedFile.ContentType. This extract the correct MIME type for all file type that I have tried (to date: txt, jpg, doc, xls) except for pdf. This return a MIME type of "application/unknown".

This continues to work for IE7 and IE8. It also work for earlier version of FireFox (versions different than reported for this issue).

Reproducible: Always

Steps to Reproduce:
using VisualStudio 2008 with ASP.Net 3.5.

1. Create new ASPX.
2. Add FileUpload control.
3. Add Button control.
4. Add Click event to Button.
5. Add break within click event.
6. Run page.
7. Browse... to PDF.
8. Submit. This should bring you to break.
9. Watch FileUpload.PostedFile.ContentType. It will report "application/unknown".
10. Repeat with other file types. MIME will be correctly reported.
