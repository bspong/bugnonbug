Long Hang Switching Column Display To 'Show All' In Google Keyword Tool
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 6.0; en-US; rv:1.9.1.2) Gecko/20090729 Firefox/3.5.2 GTB5 (.NET CLR 3.5.30729)
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 6.0; en-US; rv:1.9.1.2) Gecko/20090729 Firefox/3.5.2 GTB5 (.NET CLR 3.5.30729)

When doing keyword research with the Google Keyword Tool, FireFox 3.5.2 hang for a long period of time after switching the column view to 'Show All' from the drop-down menu. FireFox eventually resumes. Less keywords entered seems to provide shorter hang time.

Reproducible: Always

Steps to Reproduce:
1. Visit: https://adwords.google.com/select/KeywordToolExternal
2. Type a keyword (or multiple keywords) into the box (Use synonym checked), ex: firefox, internet browser (note: more keywords entered = longer hang time)
3. Fill in the Captcha, click 'Get keyword ideas'
4. For the 'Choose column to display:' dropdown menu, choose: Show All
Actual Results:
FF hang for a long time -becomes completely unresponsive.

Expected Results:
Page to refresh with full data and all column after a short time.

Tested same action in two other browsers:
1.) Internet Explorer 8.0.6001.18813
2.) Safari 4.0.2 (530.19.1)

IE refreshes the data with a short delay.
Safari refreshes the data instantly.
