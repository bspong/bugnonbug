javascript: top and left position reported wrong if parent element ha "overflow: hidden" (border-width of parent is ignored)
User-Agent: Mozilla/5.0 (X11; U; Linux i686; de; rv:1.8.1.12) Gecko/20080208 Firefox/2.0.0.12 (Dropline GNOME)
Build Identifier: Mozilla/5.0 (X11; U; Linux i686; de; rv:1.8.1.12) Gecko/20080208 Firefox/2.0.0.12

if the parent element (div in my example) ha "overflow: hidden", child node left and top position are wrong.

I used three different way to get the position:
1. web developer toolbar
2. firebug
3. quirksmode's getPos() script

All report the wrong size. Firefox 3 doe not have this problem.
Tests show me that the all script are ignoring the border-width of the parent element.

Reproducible: Always

Steps to Reproduce:
1. create clean html:

<html>
<head></head>
<body style="margin: 0">
<div style="overflow: hidden; border: 50px solid red; height: 300px;">
<p style="margin: 0; padding: 0">test</p>
</div>
</body>
</html>
2. load in Firefox 2
3. Use Javascript or Web Developer Addon or Firebug to see the position
4. Change the border-width from 50px to 1px-
5. Reported position is not being changed
Actual Results:
position of <p> is 0px for top and left

Expected Results:
position of the <p> should be: top: 50px, left: 50px
