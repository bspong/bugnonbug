Firefox icon don't change back after running the Add Printer Guide
User-Agent: Mozilla/4.0 (compatible; MSIE 7.0; Windows NT 5.1; .NET CLR 1.1.4322)
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; da; rv:1.9.0.1) Gecko/2008070208 Firefox/3.0.1

When no printer is installed on the pc, you'll get a message telling you, that you need to install a printer when choosing file > print preview.

If you click yes this will start the Add Printere Guide.
When this guide launches, the Firefox application icon change to a printer

After adding a printer through the guide and closing the guide, the firefox application icon don't change back to it's originally icon

Reproducible: Always

Steps to Reproduce:
1. Delete all printer on your Windows XP(R) computer
2. Start Firefox(TM)
3. Choose Print Preview from the File Menu
4. Chick Yes, when told, that printer related job requires a printer to be installed
5. Now notice, that when the Add Printer Guide appears, the Firefox(TM) application icon change to a printer
6. Add any printer through the guide.
7. Exit the guide.
8. Now notice, that the Firefox(TM) application icon is still a printer, even though the Add Printer Guide dialog isn't shown anymore
Actual Results:
The Firefox application icon changed to a printer icon

Expected Results:
Changed back the application icon or haven't changed it at all
