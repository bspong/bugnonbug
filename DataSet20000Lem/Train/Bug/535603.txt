Clear recent history title not set to all correctly on opening window
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9.1.6) Gecko/20091201 Firefox/3.5.6 (.NET CLR 3.5.30729)
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9.1.6) Gecko/20091201 Firefox/3.5.6 (.NET CLR 3.5.30729)

Selecting the clear recent history option open up a dialogue with the title bar set to "Clear Recent History" This title change to "Clear All History" if you select the "Everything" option for the "Time range to clear" drop down. However if you've previously cleared everything, and open the clear recent history dialogue again, Everything is automatically selected, however the window is titled "Clear Recent History" not "Clear All History" Re-selecting Everything from the drop down doe not correct this unless a non everything value is selected first.

Reproducible: Always

Steps to Reproduce:
1.ctrl+shift+del
2.Observe title
3.Select 'Everything'
4.Observe title
5.Clear history and close dialogue
6.ctrl+shift+del
7.Observe title
Actual Results:
Title say "Clear Recent History" while "Everything" is selected from the drop down menu.

Expected Results:
Tile should say "Clear All History" when "Everything" is selected from the time range drop down menu
