The scroll key graphic of the mouse remain on the screen although deactivated.
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9.0.10) Gecko/2009042316 Firefox/3.0.10
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9.0.10) Gecko/2009042316 Firefox/3.0.10

While opening any web page with more than one page view, the user is provided with a scroll feature which is active in Mozilla Firefox a well.

Here the user make the scroll key active in the page, and us the key combination "Window + D" to minimize all the active application which include the Firefox.

Now user the user again us the key combination "Window + D" to restore all the application including Firefox.

Here the user can observe that the graphic of the scroll key remain on the screen although the scroll functionality is not active in this screen.

It is also observed that the graphic remains in all others screen a well irrespective of the application including the desktop.

The only way this graphic disappears is when the user return to Mozilla Firefox and selects the scroll key again. Using scroll key in any other operation would not make this graphic disappear.


Reproducible: Always

Steps to Reproduce:
1.Open any Web page of more than one page on Mozilla Firefox.
2. Use the Scroll key ( Centre key) of the mouse to activate the scroll functionality.
3.Use the key combination "Window + D" to minimize all applications.
4.Again use the combination "Window +D" to restore all applications.
5. Observe that the graphic specific to scroll functionality is still visible on the screen although the functionality is not active.


Expected Results:
The scroll key graphic should not remain on the screen when the functionality is not active.

The software should have not enabled the scroll graphic indiacation when the Firefox ha been restored.
