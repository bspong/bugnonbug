Messenger Plus! Script Editor cause Firefox to "freeze"
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-GB; rv:1.9b5) Gecko/2008032620 Firefox/3.0b5
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-GB; rv:1.9b5) Gecko/2008032620 Firefox/3.0b5

Messenger Plus! Live version 4.60.0.324 - Released 19/03/2008, with StuffPlug 3 Disabled.

The script editor cause the Firefox interface to "freeze" when opened. For example, Firefox doe not display "Firefox" when it's switched from another window. It still display the previous window's contents, until the screen is moved.

Reproducible: Sometimes

Steps to Reproduce:
1.Download Messenger Plus! Live (http://www.msgpluslive.net/)
2.Open Script Editor ((A new menu after File, Contatcts... Help) Plus!-> Preferences -> General Tab -> Scripts (on the left hand side) -> Create New.
3.Feel free to write a script.
Actual Results:
Switching back to Firefox wa normal.

Expected Results:
Sometimes, switching back to Firefox cause the content of the Script Editor to be displayed still (like Firefox wa frozen), until mouseover some buttons, then those button will appear. Otherwise the rest of the window still consists of the Script Editor.
