Saved password for a site deleted after log in to the site using the user name and password described and in turn, log out
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9.1.2) Gecko/20090729 Firefox/3.5.2
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9.1.2) Gecko/20090729 Firefox/3.5.2

After I log in to a site using one of the account in "Saved Passwords" window without typing the full username and password, that account automatically log out after a click on a link associated with the site, and all the saved password associated with the site are deleted.

Reproducible: Always

Steps to Reproduce:
1. Make sure first that four or more password are saved for the site.
2. Log in to a site described in the "Saved Passwords" window with an account described in the same window, but without typing the full log in details.
3. Click on a link within the site itself, or two, or more.
Actual Results:
The account log out, and all the saved password for the site are deleted.

Expected Results:
Firefox must preserve the saved password for the site after a log in into that site using one of the saved password and the associated username.

This bug usually occurs, but on a limited number of sites. The computer's configuration is: ASUS P5S-MX SE motherboard, Intel Pentium E2140 @ 1600 MHz Dual-Core processor, 512 MB RAM, and DirectX 9.0c.
