Page Info Security section should show site-specific saved password
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.8.1.3) Gecko/20070309 Firefox/2.0.0.3 Creative ZENcast v1.02.10
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9a4) Gecko/20070427 GranParadiso/3.0a4

The new Page Info dialog contains a Security section which tell user whether a site ha stored cooky on the users' computer and/or ha Firefox saved a password for the site. In addition, there are the "View Cookies" and "View Saved Passwords" buttons, which I think the user expects to show site-specific cooky and saved password (instead of simply showing *all* cooky and saved passwords). However, the current implementation in FF3.0a4 is the same to the equivalent button in the Options dialog.

Reproducible: Always

Steps to Reproduce:
1. Invoke the Page Info dialog for a page.
2. Navigate to the Security section.
3. Click either View Cookies or View Saved Passwords.
