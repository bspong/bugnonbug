XUL popup position attribute before_end doe not work
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.8.1.13) Gecko/20080311 Firefox/2.0.0.13
Build Identifier: Mozilla/5.0 (X11; U; Linux i686; en-US; rv:1.9b5) Gecko/2008032619 Firefox/3.0b5

A XUL popup element with position=before_end doe not work properly. It is supposed to [1] "The popup appears above the element with the popup's lower-right corner aligned with the upper-right corner of the element. The right edge of the element and the popup are aligned." However, in FF3.0 it appears at the position (top-left of the popup) of the mouse cursor.

This (incorrect popup position) happens on all platform tested.

If, on the other hand, the mouse is positioned such that the bottom-right corner of the popup will be off the screen (only BOTH too low and too right) then the popup appears and immediately disappears again. Some testing show that the menu "is there" but not showing. Further click while it is in this "disappeared" state hit the menu item inside.

This (disappearing) seems to happen only on Linux (tested with Ubuntu 7.10).

I'm attaching an test-case extension which work in FF2.0.x but break in FF3.0b5. The XUL code concerned is:

<image id="popuptester-image" context="popuptester-popup"
src="chrome://popuptester/content/popuptester.png"
/>
<menupopup id="popuptester-popup" position="before_end">
<menuitem label="Menu filler"/>

</menupopup>
</statusbarpanel>
</statusbar>

</overlay>
