Only when logged into Yahoo mail so far
The code below seems to be the problem code. When I click on the empty option for bulk mail, a pop-in window prompt me to confirm deletion. Upon confirm, the window reappears.
Upon cancel after confirm, the window/pop in disappears and the deletion from previous confirm take effect. This is an annoying but non-critical error. May have ramification in other area involving javascript.


function EmptyFolder(folder)
{
var warnonempty = folder == "@B@Bulk" ? (!noDelBulkMsgWarning) : (!noDelAllMsgWarning)
var emptyFol = true;
if(warnonempty)
{
var sMessage = folder == "@B@Bulk" ? "Are you sure? Deleted Bulk message are gone forever, and will not go to your Trash folder." : "Are you sure you want to empty the folder?";
emptyFol = showEmptyFolderWarning(sMessage, folder);
}

if(emptyFol)
{
var url = "/ym/ShowFolder?";
url += folder == "Trash" ? "ET" : "EB";
url += "=1&.crumb=kkzlX4LdAS0&reset=1&YY=21760&y5beta=yes&y5beta=yes&inc=25&order=down&sort=date&pos=0&view=a&head=b&box=Inbox";
window.open(url, '_self');
}
}

The window to prompt deletion confirmation reappeared on "ok" consistently.
