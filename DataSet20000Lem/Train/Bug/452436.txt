Cursor disappears from mail window in Sun Java Communications Express when user attempt to spellcheck a message and then edit the message further.
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9.0.1) Gecko/2008070208 Firefox/3.0.1
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9.0.1) Gecko/2008070208 Firefox/3.0.1

After user spellcheck a message created using Sun Java's Communication Express, if you click on the edit button to continue composing your message, the cursor disappears. If you then click on the message window, the cursor is still invisible and if you type anything it appears at the beginning of the line closest to which you clicked. There is no easy way to navigate to the different part of the text to add or remove additional text. These problem do not appear to occur in FF2 or IE7.

Reproducible: Always

Steps to Reproduce:
1.After writing some text, click Spell Check.
2.Replace the appropriate words.
3.Click Edit.

Actual Results:
No cursor is visible.
The up, down key don't appear to work.


Expected Results:
Should see cursor somewhere within the text preferably at the last point of editing.
