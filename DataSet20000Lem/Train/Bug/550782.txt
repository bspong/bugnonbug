Extension update and FF restart required after every system restart - XP and Win 7, FF3.6
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US; rv:1.9.2) Gecko/20100115 Firefox/3.6
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US; rv:1.9.2) Gecko/20100115 Firefox/3.6

After a clean install on newly-formatted XP and Windows 7 machines, FF 3.6 repeatedly disables certain (but not all) extensions:
Name Version Enabled ID
QuickDrag 2.0.2.1 false quickdrag@mozilla.ktechcomputing.com
PDF Download 3.0.0.1 false {37E4D8EA-8BDA-4831-8EA1-89053939A250}
Forecastbar Enhanced 0.9.6 false {3CE993BF-A3D9-4fd2-B3B6-768CBBC337F8}

After every system restart, FF disables those three extensions, and then requires compatibility update and FF restart. After the FF restart, the extension will be enabled-- until the next system restart. The compatibility update load the same version of the extensions.

Reproducible: Always

Steps to Reproduce:
Perform once:
1. Perform clean installation of FF 3.6 on XP/SP3 or Win 7/32
2. Install extensions: QuickDrag 2.0.2.1, PDF Download 3.0.0.1, Forecastbar Enhanced 0.9.6
Repeat to reproduce:
3. Restart system

Actual Results:
1. The extension are disabled.
2. After performing compatibility upgrades, the extension are enabled until next system restart.


Expected Results:
After system restarts, the extension should remain enabled.

The three extension were properly enabled on the same machine prior to their reformatting and clean FF 3.6 installs. The Win 7 machine wa formerly Vista.
