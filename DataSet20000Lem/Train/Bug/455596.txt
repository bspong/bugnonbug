Strange Firefox behavior after some web page ha been opened in a background tab
User-Agent: Mozilla/5.0 (Macintosh; U; PPC Mac OS X 10.4; en-GB; rv:1.9.0.1) Gecko/2008070206 Firefox/3.0.1
Build Identifier: Mozilla/5.0 (Macintosh; U; PPC Mac OS X 10.4; en-GB; rv:1.9.0.1) Gecko/2008070206 Firefox/3.0.1

When the page http://www.basango.com/index.php?action=breve&id_article=1039149 ha been loaded in a background tab (i.e. it is not the tab that is currently displayed), Firefox is slow and behaves strangely (see below).

Reproducible: Always

Steps to Reproduce:
1. Open http://www.basango.com/index.php?action=breve&id_article=1039149 and other tabs. Make sure that the tab that is currently displayed doe not correspond to the given URL (otherwise the following problem do not occur).
2. Switch to another application.
3. Click on the Firefox window.
Actual Results:
* The Firefox window doesn't go to the foreground (if it wasn't already).
* The Firefox menu doesn't appear (one still ha the menu of the previous application).
* The mouse wheel ha no effect.
* When moving the mouse pointer over a link, the shape of the mouse pointer doesn't change (but the status line is updated a expected, displaying the URL of the link).
* Firefox is slow to respond to events.

Expected Results:
These problem should not occur.

When the given URL is open, Firefox take 100% CPU time.
