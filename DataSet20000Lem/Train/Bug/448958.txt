Build Identifier: Mozilla/5.0 (X11; U; Linux i686; en-US; rv:1.9.0.1) Gecko/19700101 Firefox/3.0.1

Using my laptop (Synaptics)touchpad scroll button (4-way) for issuing page back/forward request with the left/right button is my preferred usage of this hardware feature. This work fine on the same OS under FF2.
Setting this feature from the default (no action) in FF3 via the about:config setting:
mousewheel.horizscroll.withnokey.action = 2 (0 = default, 1 = horizontal scrolling) work for the first time after change then reverts to right click = back, left click = forward (the reverse of FF2's config) i.e. is reversed.

Xorg.conf:
Section "InputDevice"
Identifier "Touchpad"
Driver "synaptics"
Option "Device" "/dev/psaux"
Option "Protocol" "auto-dev"
Option "LeftEdge" "1700"
Option "RightEdge" "5300"
Option "TopEdge" "1700"
Option "BottomEdge" "4200"
Option "FingerLow" "25"
Option "FingerHigh" "30"
Option "MaxTapTime" "180"
Option "MaxTapMove" "220"
Option "VertScrollDelta" "100"
Option "HorizScrollDelta" "0"
Option "MinSpeed" "0.09"
Option "MaxSpeed" "0.18"
Option "AccelFactor" "0.0015"
Option "SHMConfig" "on"
EndSection

-works fine for FF2 - why not FF3?
