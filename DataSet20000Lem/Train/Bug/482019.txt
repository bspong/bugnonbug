Removed parameter from the xml search plugin appears anyway in the search url
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 6.0; en-US; rv:1.9.0.7) Gecko/2009021910 Firefox/3.0.7 (.NET CLR 3.5.30729)
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 6.0; en-US; rv:1.9.0.7) Gecko/2009021910 Firefox/3.0.7 (.NET CLR 3.5.30729)

I have removed from the google's xml search plugin the following parameters:
<!-- Dynamic parameter -->
<Param name="rls" value="{moz:distributionID}:{moz:locale}:{moz:official}"/>
<MozParam name="client" condition="defaultEngine" trueValue="firefox-a" falseValue="firefox"/>
This because with those parameter I get redirected to the localized google search page, instead I want to stay on google.com. But still I get those two parameter added to the search url:
...&rls=org.mozilla:en-US:official&client=firefox-a
I have tried to switch the search provider, in the case that wa cached but nothing, I've also restarted firefox but nothing. I search also in the about:config for cached informations, but even here there wa nothing.

Reproducible: Always

Steps to Reproduce:
1. Modify the google.xml search plugin, remove the two parameter rls and client
2. Try to search something with the search box in google
Actual Results:
To the search url are added two parameter which are anymore present in the search plugin xml: rls and client

Expected Results:
The search url without the deleted parameters, only the parameter in the search plugin should appears in the search url.
