Selected check box change when browsing, then using the 'Back' Button
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.7.6) Gecko/20050223 Firefox/1.0.1
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.7.6) Gecko/20050223 Firefox/1.0.1

On the page http://www.geocaching.com/seek/nearest.aspx?zip=98671&x=0&y=0,
selected checkboxes change when you click on a cache description link and then
click the 'Back' button. The check box previously selected will be blank and
the the check box three space above the one you had selected will be selected
when the page reloads.

Build information is above; the workstation is Windows XP Professional, all
recent security patch applied.

Reproducible: Always

Steps to Reproduce:
1. Load http://www.geocaching.com/seek/nearest.aspx?zip=98671&x=0&y=0
2. Check the checkbox for the fifth listed cache on the page
3. Click on the cache name link for a description of the cache
4. Click the 'Back' button


Actual Results:
The checkbox selected will now be blank; the checkbox three space above the one
you checked will be checked.

Expected Results:
Selected checkboxes should have remained selected a originally indicated by the
user.
