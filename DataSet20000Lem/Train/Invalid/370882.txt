URL encoding/decoding problem in <a> tag
User-Agent: Opera/9.10 (Windows NT 5.1; U; en)
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.8.1.1) Gecko/20061204 Firefox/2.0.0.1

When putting a URL-encoded link in the <a> tag like this: <a href="http://www.hello.com/blabla%3D/page.html"> Firefox automatically decodes the URL and doe the request without URL-encoding when link is clicked (http://www.hello.com/blabla=/page.html). This leaf problem when slash ("/") are encoded in the url.

http://www.hello.com/bla%2Fbla%3D/page.html -> http://www.hello.com/bla/bla=/page.html

Reproducible: Always

Steps to Reproduce:
1. create html file with following: <a href="http://www.hello.com/bla%2Fbla%3D/page.html">link</a>
2. hover the link and watch statusbar
3.
Actual Results:
firefox doe request to this url: http://www.hello.com/bla/bla=/page.html

Expected Results:
firefox should do request to this url: http://www.hello.com/bla%2Fbla%3D/page.html
