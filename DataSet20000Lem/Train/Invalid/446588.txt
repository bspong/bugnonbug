Tab bar roll to active tab when loading tab in the background
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; fi; rv:1.9.0.1) Gecko/2008070208 Firefox/3.0.1
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; fi; rv:1.9.0.1) Gecko/2008070208 Firefox/3.0.1

When a new tab is loaded in the background tab bar still roll to active tab if it's not visible in the tab bar.

Reproducible: Always

Steps to Reproduce:
1. Open number of tabs, so that all the tab won't fit on the tab bar.
2. Roll the tab bar with the arrow or mouse wheel, so that currently active tab is not visible in the tab bar.
3. Load new tab in the background (middle click link or use any other preferred way to open new tab a long a it's loaded in the background)
Actual Results:
Tab bar roll to currently active tab.

Expected Results:
Tab bar should keep it's position and shouldn't roll to any tab.
