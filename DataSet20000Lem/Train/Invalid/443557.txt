Babylon dictionary mistakenly highlight text inside text box
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9) Gecko/2008052906 Firefox/3.0
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9) Gecko/2008052906 Firefox/3.0

When you highlight text with Babylon (default: CTRL-right-click, used: middle mouse button) often not the word directly under the pointer is selected, but one of the word to the right of the pointer in the same line. Despite not serious, this bug is very annoying when you need to write text in other languages.

Reproducible: Sometimes

Steps to Reproduce:
1. Install Babylon
2. Start writing multiple line inside a textbox
3. Use Babylon access button to highlight word at random along the text.
Actual Results:
Babylon user are familiar with this problem.

Expected Results:
Bogus result in the Babylon dictionary.

Show translation to the selected word.
