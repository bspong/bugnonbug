Firefox 4 doe not recognize GoDaddy SSL certificate: sec_error_unknown_issuer
User-Agent: Mozilla/5.0 (Windows NT 6.1; WOW64; rv:2.0) Gecko/20100101 Firefox/4.0
Build Identifier: Mozilla/5.0 (Windows NT 6.1; WOW64; rv:2.0) Gecko/20100101 Firefox/4.0

Firefox 4 simply doesn't recognize GoDaddy a the issuer of a valid certificate.

Until recently we never had any such issues, and this is very alarming, a client are relying on the SSL connection a reliable.

When testing on 3.6 on Linux, the issuer is recognized a GoDaddy, with no problems.

The domain <a href="https://www.checker-soft.com/">checker-soft.com</a> ha it certificate set properly, <a href="http://www.sslshopper.com/ssl-checker.html#hostname=www.checker-soft.com">as can be seen in this test</a>.

This is <b>not</b> a duplicate of <a href="https://bugzilla.mozilla.org/show_bug.cgi?id=399324">bug 399324</a>. That bug describes a missing intermediate chain / certificate, <a href="http://www.sslshopper.com/ssl-checker.html#hostname=www.settrade.com">as can be seen in this test</a>.

1. Go to: https://www.checker-soft.com/
2. See the message.
3. Test the address in: http://www.sslshopper.com/ssl-checker.html#hostname=www.checker-soft.com
