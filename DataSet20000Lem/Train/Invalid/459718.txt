Enabled JavaScript on clicking a link crash PC
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; de; rv:1.9.0.3) Gecko/2008092417 Firefox/3.0.3
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; de; rv:1.9.0.3) Gecko/2008092417 Firefox/3.0.3

On certain pages, when I click a link Windows XP crash without giving any error-message. It only happens when JavaScript is enabled.

Two examples:
1a) Go to http://alice.aol.de/
1b) Click the link to https://www.alice-dsl.de/kundencenter/Start_P.do?cid=aol_hkcr

2a) Go to http://www.teltarif.de/arch/2008/kw41/s31552.html
2b) Click the link to http://www.teltarif.de/arch/2008/kw41/s31552.html?page=2

Both site just work fine with an other browser like Opera or Apple Safari. I can not reproduce this problem on any other PC. It appears to happen only with one of my machines.

Reproducible: Always

Steps to Reproduce:
1. Enable JavaScript
2. Go to http://alice.aol.de/
3. Click the link to https://www.alice-dsl.de/kundencenter/Start_P.do?cid=aol_hkcr
Actual Results:
The whole system crash and start with reboot.

Expected Results:
Follow the link to the requested page.
