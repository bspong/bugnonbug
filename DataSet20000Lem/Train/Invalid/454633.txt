Running a fully patched and updated CENTOS5 system (also tested on fully patched RHEL5 system) running two completely different nvidia video card and different version of the binary linux nvidia driver or the latest open source nv driver provided by RedHat for RHEL5 and CENTOS5. Attempting to visit this website: http://wiki.ocsinventory-ng.org/index.php?title=OCS_Inventory_NG:Documentation cause Xorg to crash and reboot itself logging the user completely out of the desktop. Tried it in both gnome and fluxbox with and without composite enabled and in both case the same result occurred.

This problem doe not occur with konqueror.

Reproducible: Always

Steps to Reproduce:
1. Use either binary nvidia driver or the nv driver and load xorg
2. Go to http://wiki.ocsinventory-ng.org/index.php?title=OCS_Inventory_NG:Documentation
3. Xorg will crash logging the user out
Actual Results:
Xorg crashed, reset itself, and logged the user out

Expected Results:
Xorg should not crash and should not log the user out
