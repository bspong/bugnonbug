dns-prefetch hint broken when protocol is not included
User Agent: Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/535.1 (KHTML, like Gecko) Chrome/13.0.782.32 Safari/535.1

Steps to reproduce:

Try the following page, monitoring DNS request using a DNS proxy or using a packet sniffer:

<html>
<head>
<link rel="dns-prefetch" href="www.somefakedns.com">
</head>
<body>
Hi
</body>
</html>


Actual results:

No DNS request wa issued for www.somefakedns.com. It is only requested if a protocol is included with the dns prefetch hint.


Expected results:

A DNS request should have been issued for www.somefakedns.com.
