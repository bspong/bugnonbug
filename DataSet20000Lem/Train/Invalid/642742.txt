Facebook - Badges - "Remove this post" remove all post, re-posting brings them back
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US) AppleWebKit/534.16 (KHTML, like Gecko) Chrome/10.0.648.133 Safari/534.16
Build Identifier: FF4, IE 9.0.8080.16413, Chrome 10.0.648.133

when removing a post from one's own wall, all FF4 post are removed. re-posting once again brig the remaining post back to the wall. in IE9, when all the post are removed from own wall, they are also removed from friend wall (and are re-appeared when user re-posts to own wall).

Reproducible: Always

Steps to Reproduce:
1) Make several post to of badge to wall.
2) From wall, select "remove post" in the post option X.
3) Refresh the page.
Actual Results:
All post are removed from wall, not only the one that wa selected.
--> Go back to FF4 badge page and post a new badge to wall. -> The remaining badge are posted back on the wall with the new one. see attached video.
NOTE: when posting and removing the post in IE9, all the post disappear and re-appear on friend's wall a well.

Expected Results:
Only the removed badge should be removed from wall.
