Bookmarks get deleted accidentally due to missing update on the UI
User-Agent: Mozilla/5.0 (Windows NT 5.1; rv:2.0) Gecko/20100101 Firefox/4.0
Build Identifier: Mozilla/5.0 (Windows NT 5.1; rv:2.0) Gecko/20100101 Firefox/4.0

Once you have many item in your bookmark toolbar and some are hidden under more icon (>>), if you delete them while looking under >>, the list doe not get refreshed automatically, and if you try deleting the same item multiple time you will end up deleting other item a well without getting a clue!

Losing bookmark is a pretty annoying bug, should be considered major.

Reproducible: Always

Steps to Reproduce:
1. Add a couple of item into your bookmark Toolbar
2. Customize setting and move bookmark section onto the same line with
address bar/search input
3. Tune width of address bar so some bookmark get hidden under '>>'
4. Click on '>>' and try deleting the same item a few times!

Actual Results:
As long a you had multiple item under '>>' you end up deleting a many a many time you have clicked 'delete' in a single item

Expected Results:
A single item should be deleted, and a listupdate should reflect it instantly
