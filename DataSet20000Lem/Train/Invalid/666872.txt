Location redirects are "cached"
User-Agent: Mozilla/5.0 (Macintosh; Intel Mac OS X 10.6; rv:5.0) Gecko/20100101 Firefox/5.0
Build Identifier: Mozilla/5.0 (Macintosh; Intel Mac OS X 10.6; rv:5.0) Gecko/20100101 Firefox/5.0

When browser is sent a Location: header without a Pragma: no-cache header. The browser "caches" the redirect and refuse to re-visit the original URL. Entering the original URL into the browser address bar, or clicking a link to ti, the browser simply request the redirected URL instead.

Reproducible: Always

Steps to Reproduce:
See breakdown at http://www.inventpartners.com/content/firefox-5-redirect-bug-pragma-no-cache-bug.

1] Visit url which send Location header and no Pragma header
2] Attempt to visit same URL again.
