Minefield lock up while flash video is loading/playing.
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US; rv:1.9.3a1pre) Gecko/20100129 Minefield/3.7a1pre
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US; rv:1.9.3a1pre) Gecko/20100129 Minefield/3.7a1pre

This lock up occurs when a flash video is loading or playing. I have been able to get Minefield to lock up by clicking the fullscreen video tab, and by clicking a java link in a seperate tab while flash video is playing in a minimised tab.

The lock up cause the interface of Minefield to be come unresponsive, however audio continues to play. The only way to close out of Minefield is to kill the thread via the task manager.

I have found 2 way to work around the crash:
1. Open the taskmanager and terminate the thread "mozilla-runtime.exe" running in a a 32-bit thread.
2. Mousing over the start bar icon for Minefield and waiting for the tab preview to load. Clicking on the preview cause the interface to update (changing tabs) and Minefield continues to operate a intended.

While filling out this bug report I re-opened the flash video to replicate the issue and am now unable to leave the current tab without using the mouse-over window preview trick. Possibly unrelated is that after using the trick, my mouse scroll fails to work. My click register on tab clicked (the close x becomes visible) but the tab doe not change. In order to scroll I have to use the scroll bar and drag it, clicking the directional arrow on the scroll bar ha no affect.

The lock up occurs regardless of whether there are addons on or not, but I have included the debug info in the additional information for when I have addons enabled (and addon checking disabled, although this doe not affect the result).

Reproducible: Sometimes

Steps to Reproduce:
1. Open a youtube video such a the one provided in this report.
2. Play the flash video.
3. Open the piratebay.org and sort the torrent by seeders.
4. Minefield should now lock up.
Actual Results:
The interface lock up.

Expected Results:
Continued to work a intended.
