NOTE: This bug includes and broadens all of the following:
Bug#329853, Bug#318810, Bug#330206

Every time the Organize (Bookmarks, History) function open the PLACES UI, it appears in a fairly useless top-level view.

Reproducible: Always



Expected Results:
What it should do:
The UI should look just like it did the last time it wa closed. All of the following "state" of the UI should be remembered. Between session is a "nice-to-have," within a session is important.

1. Location of the pop-up;
2. Size of the pop-up (IMPORTANT);
3. Position of the split (Bug#330206);
4. What folder in the left-hand tree view are expanded (Bug#329853, enlarged);
5. Sort of bookmark and folder (Bug#318810).
