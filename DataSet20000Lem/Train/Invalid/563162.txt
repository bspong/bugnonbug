When creating new folder on desktop folder text unselected
User-Agent: Mozilla/5.0 (Macintosh; U; Intel Mac OS X 10.6; en-US; rv:1.9.2.3) Gecko/20100401 Firefox/3.6.3
Build Identifier: Mozilla/5.0 (Macintosh; U; Intel Mac OS X 10.6; en-US; rv:1.9.2.3) Gecko/20100401 Firefox/3.6.3

When running Firefox, creating a new folder on the desktop (Finder) the folder text is unselected and by default named: untitled folder.
Default in OSX is when creating a new folder the folder text is selected for entering a folder name.

Reproducible: Sometimes

Steps to Reproduce:
1. Run Firefox. Keeping the Firefox window small enough to see the desktop.
2. Right click on the desktop, select New Folder.
3. A new folder is created on the desktop and named untitled folder but the folder text is not selected for entering a folder name.
Actual Results:
A new folder is created on the desktop and named untitled folder but the folder text is not selected for entering a folder name.

Expected Results:
As by default OSX - folder name stay selected for entering a folder name.

This bug showed up in Firefox 3.6 and forward (3.6.3).
