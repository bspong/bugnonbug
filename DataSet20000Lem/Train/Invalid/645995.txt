UI is not consistent between Private Browsing started from UI and command line
User-Agent: Mozilla/5.0 (Windows NT 5.1; rv:2.0) Gecko/20100101 Firefox/4.0
Build Identifier: Mozilla/5.0 (Windows NT 5.1; rv:2.0) Gecko/20100101 Firefox/4.0

When Private Browsing is started from UI the menu button change from an orange colour to purple and back to orange when stopped. But when Private Browsing is started from the command line the menu button remains an orange colour and the Start Private Browsing menu option is greyed out (not selectable)

Reproducible: Always

Steps to Reproduce:
1. Open a "Command Prompt" from window start menu
2. Change directory to where Firefox is installed. For example, cd "c:\program files\mozilla firefox\"
3. Start firefox with private browsing enabled by typing the following command: firefox.exe /private
Actual Results:
Firefox start with private browsing enabled but menu button is an orange colour

Expected Results:
Firefox start with private browsing enabled but menu button is a purple colour
