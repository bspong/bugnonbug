Icons in firefox prefs not compliant with theme (problem in high contrast)
User-Agent: Mozilla/5.0 (X11; U; Linux i686; en-US; rv:1.7.8) Gecko/20050607 Firefox/1.0.4
Build Identifier: Mozilla/5.0 (X11; U; SunOS sun4u; en-US; rv:1.9a1) Gecko/20060416 Firefox/3.0a1

User set different theme on his desktop preferences, icon in preference setting dialog are not compliant with gnome theme settings. This will take inconvenient for users.

Reproducible: Always

Steps to Reproduce:
1. Change gnome theme settings, select Launch bar->Desktop->Preferences->Theme,
select High ContrastInverse theme, then click Theme Details button. Select
Controls tab and set HighContrastLargePrintInverse, select Icon tab and set
HighContrastInverse. Then close these two setting dialog.
2. Launch firefox and open preference dialog.
Actual Results:
Icons of different preference tab are not compliant with gnome theme settings.

Expected Results:
These icon should be compliant with gnome theme setting.

This bug can be seen on vermillion_09/snv_36 with Mozilla/5.0 (X11; U; SunOS sun4u; en-US; rv:1.9a1) Gecko/20060416 Firefox/3.0a1.
