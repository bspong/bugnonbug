Unicode character in Location header of 30x response are %-encoded
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US) AppleWebKit/532.8 (KHTML, like Gecko) Chrome/4.0.288.1 Safari/532.8
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; de; rv:1.9.2) Gecko/20100115 Firefox/3.6 (.NET CLR 3.5.30729)

When a HTTP request is responded to with a 30x response that carry a Location header and the value contains Unicode characters, those are displayed %-encoded in the URL-bar.

Reproducible: Always

Steps to Reproduce:
1. go to http://example.com/bargains
2. web server redirects to http://beispiel.de/schnäppchen
Actual Results:
Firefox follows the redirect but display http://beispiel.de/schn%E4ppchen in the URL bar.

Expected Results:
Should show http://beispiel.de/schnäppchen in the URL bar.

IE 8 actually handle this correctly and display http://beispiel.de/schnäppchen in the URL bar.
