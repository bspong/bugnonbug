Tabbing through form elements is broken
User-Agent:       Mozilla/5.0 (Macintosh; U; Intel Mac OS X 10_6_2; en-US) AppleWebKit/532.8 (KHTML, like Gecko) Chrome/4.0.288.1 Safari/532.8
Build Identifier: Firefox 3.5.7

When using the TAB key on the keyboard to move to the next form element firefox ignores non-text input boxes. Including: select, radio and checkboxes

Reproducible: Always

Steps to Reproduce:
1.Open and View Form
2. Use the TAB key to move though the form
3.Enjoy
Actual Results:  
The select, radio and checkboxes are completely ignored and the focus goes to the next text element instead.

Expected Results:  
tab to next form element textbox -> dropdown -> checkbox and all for easy keyboard data entry

this is a deal breaker on OSX for many business users.
