Upgrade to 3.5.5 broke ALT+<any key> functionality
User-Agent:       Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9.1.5) Gecko/20091102 Firefox/3.5.5 (.NET CLR 3.5.30729)
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9.1.5) Gecko/20091102 Firefox/3.5.5 (.NET CLR 3.5.30729)

Verified on system that for all other applications the ALT key is functional.

When running Firefox, using the ALT key no longer results in any system response.  Depressing the ALT key will not activate the menu bar.  Issue was discovered while attempting to back track viewing history by using ALT+left arrow.

Reproducible: Always

Steps to Reproduce:
1. Run Firefox
2. Attempt to return to previous page using ATL+left arrow.
3. Attempt to access menu bar using ALT as starting point.
Actual Results:  
No response by Firefox to the ALT key input

Expected Results:  
1.
2. Firefox should display previously viewed page.
3. First entry in the menu bar should have become activated.
