scrolling DIV element not repainted on hitting "end" in Linux

The contents of a certain DIV (apparently those with "overflow scrolling") are not repainted off the screen when the screen is scrolled using using the "end" keyboard button.  

Reproducible: Always

Steps to Reproduce:
1.  open the page the link: http://jorge.ulver.no/2008/08/06/acer-aspire-one-tips-and-tricks/#AddingyourownDesktop-icons   
2. Make sure the DIV beginning with the text [Desktop Entry] is visible below the anchor.
2. Press "End" to go to the bottom of the screen

Actual Results:  
The DIV contents are not scrolled, but instead overlap other content on the page.

Expected Results:  
the DIV content should not be visible

I verified this on four different Linux PCs with diferent hardware, different distributions and different numbers of monitors (1 or 2) versions ranging from FF 3.0.1 release to Sept 13 nightly (Gecko/20080913020424 Minefield/3.1b1pre).  I could not produce it in Windows or Mac versions.
