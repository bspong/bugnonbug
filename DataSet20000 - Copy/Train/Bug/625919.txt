nstar.com, print to file creates unreadable pages 2 to n
User-Agent:       Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US; rv:1.9.2.13) Gecko/20101203 Firefox/3.6.13
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US; rv:1.9.2.13) Gecko/20101203 Firefox/3.6.13

When printing to a file, XPS or PDF, Page 1 is readable however, pages 2 to N are not readable.





Reproducible: Always

Steps to Reproduce:
1. Login to NStar.com
2. Select Bill Summary from the Left of NStar.com
3. Select Printer-Friendly Bill from the Left of NStar.com
4. from FireFile, File, Print, then Select Microsoft XPS Document Writer (make sure Print to File is not checked)
5. Click OK
6. Select the .xps filename and destination
7. From Windows Explorer, open the file by double clicking on the file
8. notice Page 1 is readable, however Pages 2 to n are not readable


Actual Results:  
unreadable pages

Expected Results:  
readable pages

The unreadable pages also occur when printing using pdfFactory and FoxIt PDF Creator.

When using FireFox 3.6.13 with other websites to print multiple pages documents, then the pages are readable.

When using IE 8.0 on nstar.com and other websites then the files are readable using all 3 methods.
