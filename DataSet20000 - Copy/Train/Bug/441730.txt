visual artifact is left when dragging a toolbar bookmark and dropping it in the same place
User-Agent:       Mozilla/5.0 (Macintosh; U; PPC Mac OS X 10.4; en-US; rv:1.9) Gecko/2008061004 Firefox/3.0
Build Identifier: Mozilla/5.0 (Macintosh; U; PPC Mac OS X 10.4; en-US; rv:1.9) Gecko/2008061004 Firefox/3.0

When you attempt to relocate a bookmark in the toolbar to a new position within the toolbar, an icon appears to indicate where the bookmark will be relocated when you release the mouse button.  This icon should disappear when you release the mouse button.  However, if you release the mouse button with the bookmark in the same position, the icon remains.

Reproducible: Always

Steps to Reproduce:
1.  Create a toolbar bookmark or use an existing one.  
2.  Press and hold the mouse button on top of a bookmark in the toolbar in order to drag it to a new place.  An icon appears to indicate where the bookmark will be relocated when you release the mouse button.  
3.  Instead of moving the position of the bookmark, release the mouse button with the bookmark in the same position.
Actual Results:  
the icon that indicate where the bookmark will be relocated after dragging and dropping does not disappear.

Expected Results:  
the icon that indicate where the bookmark will be relocated after dragging and dropping should disappear.

no theme used.  no crash caused.  no special circumstances.
