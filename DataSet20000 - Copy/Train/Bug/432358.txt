Proportional column width in HTML tables not working
User-Agent:       Mozilla/5.0 (Macintosh; U; PPC Mac OS X 10.4; en-US; rv:1.9b5) Gecko/2008032619 Firefox/3.0b5
Build Identifier: Mozilla/5.0 (Macintosh; U; PPC Mac OS X 10.4; en-US; rv:1.9b5) Gecko/2008032619 Firefox/3.0b5

My HTML code is:

<html>
    <body>
        <table cellpadding="0" cellspacing="0" rules="0" width="100%%" 
                style="font-family:  verdana;
                       table-layout: fixed; 
                       font-size:    40px"> 
                    
            <colgroup><col width="100"> </colgroup>  
            <colgroup><col width="1*">  </colgroup>
            <colgroup><col width="300"> </colgroup>
            <colgroup><col width="1*">  </colgroup>
            <colgroup><col width="100"> </colgroup>
        
            <tbody>
                <tr style="line-height:600px;text-align:center"> 
                    <td style = 'background-color:green'>
                        1
                    </td>
                    <td style = 'background-color:yellow'>
                        2
                    </td>
                    <td style = 'background-color:blue'>
                        3
                    </td>
                    <td style = 'background-color:yellow'>
                        4
                    </td>
                    <td style = 'background-color:red'>
                        5
                    </td>
                </tr>
            </tbody>
        </table>
    </body>
</html>

The width of columns 1, 3 and 5 should always be the same regardless of the size of the window. The width of columns 2 and 4 can vary depending on the window size.

If you vary the window size you can see how the width of column 1 changes. It should not.

 

Reproducible: Always

Steps to Reproduce:
1. Save the HTML as a file.
2. Open the file in Firefox 3.0
3. Resize the Firefox window.
4. Notice how the width of columns 1, 3, 5 changes.
Actual Results:  
The width of columns 1,3 and 5 changes depending on the window size.

Expected Results:  
I was expecting that the column width of columns 1 and 5 was 100 pixels regardless of the window width.
Ditto for column 3. Its width is not 300 pixels.

Proportional column width is explained in
http://www.w3.org/TR/html401/struct/tables.html#h-11.2.4.4

The column width for these tables is correctly rendered in FF 2.0.
