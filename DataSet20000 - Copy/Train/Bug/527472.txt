Ctrl tab preview search should ignore accent marks and Ñ
User-Agent:       Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9.2b1) Gecko/20091029 Firefox/3.6b1 (.NET CLR 3.5.30729)
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9.2b1) Gecko/20091029 Firefox/3.6b1 (.NET CLR 3.5.30729)

My suggestion is to ignore accent marks in letters when searching for tabs.
For example, if I have this Wikipedia page open with 10 other tabs, searching for Hernan will not find it. I'd have to search for 'Hernán Cortés' and there's no easy way of creating accent marks on a U.S. keyboard. 
http://en.wikipedia.org/wiki/Hernán_Cortés

Copied from my post at 
http://forums.mozillazine.org/viewtopic.php?f=23&t=1572095&p=7953235#p7953235

Reproducible: Always

Steps to Reproduce:
1. Enable tab previews. browser.ctrlTab.previews to true via about:config
2. Open up three tabs.
3. Open up this page http://en.wikipedia.org/wiki/Hernán_Cortés (or any page that has accent marks or non-US characters.
4. use Ctrl+Shift+Tab to show the Ctrl tab preview
5. Search for Hernan Cortes (note, not using any accents)
Actual Results:  
No results are shown.

Expected Results:  
The Hernán Cortés tab should be selected.
