User-Agent:       Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.8.0.3) Gecko/20060426 Firefox/1.5.0.3
Build Identifier: Firefox/1.5.0.3

If there is a string that is 4890 characters or greater within a single xml node it will not render.
If you take out one character from this string, it works.

The weird thing is that this number is not always the same, we found different examples with slightly greater and lesser numbers to recreate the same issue.  But in this case (with my attached files) that is what it gets to.

the node comes back as <foofoo>... instead of <foo>lots/of/stuff/in/a/unbroken/string</foo>

here is the surprising part, which leads us to believe it is a mozilla bug... if you remove the xsl style sheet declaration from the xml file, and then render it in firefox, it works... but NOT when you view source... when you source, you get the same <foofoo> node in the source, even though you see the full string in the browser.

that led us to believe that something that process the xml into view source is doing the same thing as a straight inline xsl transform.

Reproducible: Always

Steps to Reproduce:
1.load the xml file w/ the xsl declaration in place view in browser
2.breaks as described.
3.remove 1 char from the string... reload
4.works now.
