subscribe.css refers to feed icons which no longer exist in default theme
User-Agent:       Mozilla/5.0 (Windows; U; Win98; en-US; rv:1.8.1a3) Gecko/20060602 BonEcho/2.0a3
Build Identifier: Mozilla/5.0 (Windows; U; Win98; en-US; rv:1.8.1a3) Gecko/20060602 BonEcho/2.0a3

subscribe.css (used to apply styling to an RSS feed document) refers to feed icons in browser/skin/places, but the places directory has been removed from the default theme in this build (bonecho_2.0a3 2006-06-02-0848).

Here's the relevant code in subscribe.css:

#feedSubscribeHandleText {
  -moz-padding-start: 18px;
  background: url("chrome://browser/skin/places/livemarkItem.png") left 0 no-repeat;
}

#feedSubscribeLink {
  background: url("chrome://browser/skin/places/livemarkItem.png") left 0 no-repeat;
  -moz-padding-start: 18px;
  font-weight: bold;
  -moz-margin-end: .4em;
}



Reproducible: Always

Steps to Reproduce:
1.  Start Firefox using default theme.
2.  Go to this URL http://www.nytimes.com/
3.  Click the feed icon in the URL bar
4.  In the feed summary page which is shown, icons are missing from the "reader selection area" at the top of the page.

Actual Results:  
Icons are not shown.

Expected Results:  
Presumably, the UI designers intend for feed icons to display in the "reader selection area" at the top of the page.
