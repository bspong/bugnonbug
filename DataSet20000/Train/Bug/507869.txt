Xulrunner-1.9.1.1-r1 forces installation of unwanted sound software
User-Agent:       Mozilla/5.0 (X11; U; Linux x86_64; en-US; rv:1.9.0.11) Gecko/2009080121 Gentoo Firefox/3.0.11
Build Identifier: Gentoo Linux : net-libs/xulrunner-1.9.1.1-r1 for www-client/mozilla-firefox-3.5.1-r1 (won't compile)

To install Firefox 3.5.1 on my Gentoo system, I need to install Xulrunner 1.9.1.1, which requires in turn that I install 2 sound packages, media-libs/alsa-lib-1.0.20-r1 & media-sound/alsa-headers-1.0.20 ; Xulrunner won't compile if I use the 'nodeps' flag, giving the configuration error lines :

  checking for snd_pcm_open in -lasound... no
  configure: error: Ogg support on Linux requires the alsa library

Xulrunner 1.9.0.11 for Firefox 3.0.11 does not require sound software.

I have no speakers attached to my machine & don't have sound enabled in the kernel; I don't need any sound software in my system.  Some users are deaf
(not me) & have no conceivable use for sound software.

I reported the problem to Gentoo bugzilla & was told it was caused upstream
& to report it there, which I am doing.

Please fix Xulrunner so that it does not require sound libraries.


Reproducible: Always

Steps to Reproduce:
1. Emerge Xulrunner 1.9.1.1 on a Gentoo Linux system.
2.
3.
Actual Results:  
Xulrunner fails with a configuration error without sound packages installed.

Expected Results:  
Xulrunner will configure correctly without needing sound, as did 1.9.0.11.
