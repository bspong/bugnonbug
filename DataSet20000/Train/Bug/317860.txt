Firefox locks up after cutting (Ctrl-x) a folder in Bookmark Manager
User-Agent:       Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.8) Gecko/20051111 Firefox/1.5
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.8) Gecko/20051111 Firefox/1.5

The Bookmarks Manager provides the capability to cut the selected bookmarks/folders and then paste them in a new location. Frequently, the Bookmarks Manager stops responding before the cut bookmarks/folders can be pasted in a new posision. Then the cut bookmarks/folders are lost.

Reproducible: Always

Steps to Reproduce:
1.Start the Bookmarks Manager by selecting "Manage Bookmarks..." from the Bookmarks menu.
2. Select one or more bookmarks or folders using your mouse.
3. From the Edit menu, select Cut (or press Ctrl-X).
4. Using your mouse select a bookmark (or a folder).
5. From the Edit menu, select Paste (or press Ctrl-V)

Actual Results:  
Frequently, the Bookmarks Manager stops responding after the selected bookmarks have been cut, before they can be pasted in a new position.
The cut bookmarks/folders are lost!

Expected Results:  
The cut bookmarks/folders should be pasted before the specified bookmark/folder.
 

This bug happens frequently but not every time.
