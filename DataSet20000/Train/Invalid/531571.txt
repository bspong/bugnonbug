Middle-click in recent closed tabs cause the link to go instead loading [caused by TabMixPlus]
User-Agent:       Mozilla/5.0 (Windows; U; Windows NT 5.1; fr; rv:1.9.1.4) Gecko/20091016 Firefox/3.5.4
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 5.1; fr; rv:1.9.1.4) Gecko/20091016 Firefox/3.5.4

In history menu, you can middle-click on a link and it opens in a new tab.
In recent closed tabs, if you middle-click on a link, it disappears from the list and won't open any tab.


Reproducible: Always

Steps to Reproduce:
1. Open history menu
2. Open recent closed tabs submenu
3. Click in any link into this submenu
Actual Results:  
1. Link disappears from the submenu
2. Url is not loaded into a new tab

Expected Results:  
1. Link disappears from the submenu
2. Url is loaded into a new tab
