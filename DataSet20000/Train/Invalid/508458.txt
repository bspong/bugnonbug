Extra 1px whitespace around submit button when using display:inline
User-Agent:       Mozilla/5.0 (X11; U; Linux i686; en-US; rv:1.9.0.12) Gecko/2009070811 Ubuntu/9.04 (jaunty) Firefox/3.0.12
Build Identifier: Mozilla/5.0 (X11; U; Linux i686; en-US; rv:1.9.0.12) Gecko/2009070811 Ubuntu/9.04 (jaunty) Firefox/3.0.12

When an submit button is displayed as inline text, there is an extra 1px whitespace surrounding the element, making it larger than a DIV element rendered the same way.

Reproducible: Always

Steps to Reproduce:
You can reproduce the effect using this HTML source code:
<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

<html>

<head>
 <title>Title</title>
 <link href='style.css' rel='stylesheet' type='text/css'>
</head>

<body>

<div>a</div><input value='a' type='submit'><br>
<input value='a' type='submit'><div>a</div><br>


</body>

</html>

Here is a CSS style sheet which highlights the behavior:
* { border:0; padding:0; margin:0; outline:0; }
div { display:inline; background-color:#cfc; }
input { display:inline; background-color:#caf; font:inherit; }
Actual Results:  
You should see a square checkerboard pattern: The lavender INPUT elements larger than the green DIV elements.

Expected Results:  
Both elements should have the same dimensions when rendered the same way: as inline text.

Is there anything I'm doing wrong? I've reset every source of whitespace that I'm aware of. Are there any other properties which should be cleared as well?
