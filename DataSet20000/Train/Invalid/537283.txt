when "bumping" (to make it resize to half of screen) firefox window to right side of screen, search bar suggestions appears in different area of screen
User-Agent:       Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US; rv:1.9.1.4) Gecko/20091016 Firefox/3.5.4
Build Identifier: Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US; rv:1.9.1.4) Gecko/20091016 Firefox/3.5.4

Using windows 7 "bump" feature to move resize a window to half of the screen, the suggestions for the search bar appear in a different location of the screen. The location is always the same place no matter where the firefox window started. Bumping the window to the left side of the screen the suggestions appear where they should be. Screenshot has been attached to the bug report.

Reproducible: Always

Steps to Reproduce:
1.Launched firefox
2.'Bumped' window to right side of screen
3.Typed in search bar
Actual Results:  
Suggestions appeared in different area of the screen

Expected Results:  
Suggestions should have appeared directly under search bar

Running windows 7 on an HP Pavilion dv6338se. Successfully reproduced bug using different search providers (yahoo, wikipedia), also tried using ebay search provider, which does not provide suggestions, only search history and was still able to reproduce bug.
