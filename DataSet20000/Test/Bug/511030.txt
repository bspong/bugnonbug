Cursor position is misplaced when we try to select an empty text range following a BR
User-Agent:       Mozilla/5.0 (X11; U; Linux i686 (x86_64); en-US; rv:1.9) Gecko/2008052912 Firefox/3.0
Build Identifier: Mozilla/5.0 (X11; U; Linux i686 (x86_64); en-US; rv:1.9) Gecko/2008052912 Firefox/3.0

The cursor position is misplaced when we create and select a text range from the contents of an empty text node that follows a BR.

Reproducible: Always

Steps to Reproduce:
Click on the following plexode links to find the difference between actual and expected results.
Actual Results:  
The plexode demo given below shows this behavior. The cursor should have been around the text node but was placed at the beginning of the text node preceding the BR.
