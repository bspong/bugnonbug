User-Agent:       Mozilla/5.0 (X11; U; Linux i686; en-GB; rv:1.9.2.13) Gecko/20101206 Ubuntu/10.04 (lucid) Firefox/3.6.13
Build Identifier: Mozilla/5.0 (X11; U; Linux i686; en-GB; rv:1.9.2.13) Gecko/20101206 Ubuntu/10.04 (lucid) Firefox/3.6.13

In the Preferences-Privacy dialogue box, I have custom settings.  I left "Clear history when Firefox closes" unchecked.  My history kept clearing.  I went back and checked "Clear history when Firefox closes" changing the settings to Offline web site data and none of the other options.  Now my history is there when I start a new session.  Weird but true.

Reproducible: Always

Steps to Reproduce:
I've only had the problem with history clearing for some months.  I don't know why it happened or how you could reproduce it but I found the workaround above and tested once to see if it worked.
Actual Results:  
The workaround worked the one time I tried it.

Expected Results:  
If "Clear history when Firefox closes" is unchecked then Firefox should not clear history but for some months whenever I used Firefox my history was not there.


I went to Help and got troubleshooting information with the workaround settings in place.  This is it pasted from clipboard.  If you'd like anything else, please e-mail me, but be aware that I need beginner's style instructions.
