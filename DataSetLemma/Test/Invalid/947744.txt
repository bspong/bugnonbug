Holly crash almost instantly when OMTC is enabled and layers.use-deprecated-textures=false
Created attachment 8344385
WinDBG Log File

I wa playing around with some configuration option in the latest build of Holly, mainly trying to force OpenGL to be used instead of DirectX (not even sure that is possible on Windows these days) and I found a combination that cause Holly to insta-crash on startup.

I have the following option set in my configuration:
layers.acceleration.force-enabled=true
layers.async-video.enabled=true
layers.offmainthreadcomposition.animate-opacity=true
layers.offmainthreadcomposition.animate-transform=true
layers.offmainthreadcomposition.enabled
layers.use-deprecated-textures=false
layers.prefer-opengl=true

Setting layers.prefer-opengl back to false ha no affect on this issue and once I set layers.use-deprecated-textures=true, the crash is gone and Holly work like normal. If I disable OMTC, setting layers.use-deprecated-textures=false doe not cause a crash.
